Welcome to the LLM Communication Framework

This framework helps developers structure communication between large language models (LLMs) and code. It offers a set of vocabulary definitions designed to facilitate LLM-to-LLM interactions using JSON-LD format.

By providing a clear structure for defining the types of data being sent and received, this project aims to make the communication between systems and LLMs more transparent and effective.

Explore Vocabulary Definitions

Key Features

  • Standardized vocabulary for LLM-to-LLM communication
  • JSON-LD format for improved context and understanding
  • Extensible framework for custom application needs
  • Clear documentation and examples

Vocabulary Definitions

Types

Conversation

A sequence of messages exchanged between two or more participants (Agents or Users) in a coherent dialogue.

Message

A single unit of communication within a conversation, containing content sent from one participant to another.

Agent

An AI entity capable of sending and receiving messages in a conversation.

User

A human participant in the conversation, capable of sending and receiving messages.

Properties

id

A unique identifier for an entity (Conversation, Message, Agent, or User).

Expected value: A string that uniquely identifies the entity.

type

Specifies the type of an entity.

Expected value: One of "Conversation", "Message", "Agent", or "User".
