Welcome to the LLM Communication Framework

This framework helps developers structure communication between large language models (LLMs) and code. It offers a set of vocabulary definitions designed to facilitate LLM-to-LLM interactions using JSON-LD format.

By providing a clear structure for defining the types of data being sent and received, this project aims to make the communication between systems and LLMs more transparent and effective.

Vocabulary Definitions

Types

Conversation

A sequence of messages exchanged between two or more participants (Agents or Users) in a coherent dialogue.

Message

A single unit of communication within a conversation, containing content sent from one participant to another.

Agent

An AI entity capable of sending and receiving messages in a conversation.

User

An AI entity capable of sending and receiving messages in a conversation.

Properties

id

Definition: A unique identifier for an entity (Conversation, Message, Agent, or User). Expected value: A string that uniquely identifies the entity.

type

Definition: Specifies the type of an entity. Expected value: One of "Conversation", "Message", "Agent", or "User".

conversation

Definition: References the conversation to which a message belongs. Expected value: The id of a Conversation.

messages

Definition: An ordered list of messages that make up a conversation. Expected value: An array of Message objects.

sender

Definition: The entity (Agent or User) that sent a message. Expected value: An Agent or User object.

receiver

Definition: The entity (Agent or User) that a message is intended for. Expected value: An Agent or User object.

content

Definition: The main text content of a message. Expected value: A string containing the message text.

timestamp

Definition: The date and time when a message was sent or a conversation was started. Expected value: An ISO 8601 formatted date-time string.

sequenceNumber

Definition: A number indicating the position of a message within a conversation. Expected value: A positive integer.

metadata

Definition: Additional, context-specific information about a message or conversation. Expected value: An object containing key-value pairs of relevant metadata.

name

Definition: The name or identifier of an Agent or User. Expected value: A string representing the name.

interactionType

Definition: Categorizes the nature or purpose of a message within the conversation. Expected value: A string describing the type of interaction (e.g., "Greeting", "Question", "Response", "Conclusion").
